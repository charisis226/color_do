# 작업 목록

## 기본 지침

1. **작업 진행 순서**
   - 모든 단위 작업完成后 테스트까지 마무리
   - 테스트 완료 후 `flutter analyze`로 코드 검증
   - git 커밋 및 GitHub push完后 WORK_LOG.md 업데이트

2. **커밋 메시지 규칙**
   - `Feat:` 새로운 기능 추가
   - `Fix:` 버그 수정
   - `Docs:` 문서 추가/수정
   - `Refactor:` 코드 리팩토링

3. **작업 완료 처리**
   - WORK_LOG.md의 `진행 중` → `완료된 작업`으로 이동
   - 커밋 해시 및 완료 날짜 기록

## 사용 가이드

### 새 작업 추가
1. 위 표에서 새 번호 할당
2. 작업 내용 작성
3. 진행 중으로 상태 변경 시 `진행 중`으로 표시
4. 완료 시 `완료된 작업` 테이블로 이동

### 작업 상태 관리
- `대기`: 아직 시작하지 않음
- `진행 중`: 현재 작업 중
- `완료`: 작업 완료
